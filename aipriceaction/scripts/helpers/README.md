# scripts/helpers/

Utility scripts and CSV snapshots for fetching Vietnamese gold price data and accessing protected chart APIs.

## Scripts

### `gold-price.py`

Multi-source Vietnamese gold price fetcher. Fetches data from CafeF (bulk historical) and SJC (per-date) APIs with no external dependencies (uses only stdlib).

```bash
# CafeF SJC gold bar history (~500 days)
python3 gold-price.py cafef-sjc

# CafeF gold ring history (~180 days)
python3 gold-price.py cafef-ring

# SJC today's price (all branches)
python3 gold-price.py sjc

# SJC on a specific date
python3 gold-price.py sjc --date 2025-01-15

# SJC date range for a specific branch
python3 gold-price.py sjc --date-range 2024-01-01:2024-12-31 --branch "Hồ Chí Minh"

# Output as OHLCV candles instead of raw buy/sell
python3 gold-price.py cafef-sjc --format ohlcv

# Batch download SJC prices from 2016 to now (resumable)
python3 gold-price.py sjc-batch --output sjc-batch.csv
```

### `fetch-gold-prices.sh`

Shell wrapper that runs all `gold-price.py` commands and saves CSV snapshots into this directory. Run it to refresh all snapshot files at once.

```bash
bash scripts/helpers/fetch-gold-prices.sh
```

### `pricedancing-token.py`

Proof-of-work token miner for PriceDancing.com chart API. Reverse-engineered from the site's client-side JS:

1. Request without token → server returns 403 with a challenge string
2. Mine a nonce so that `SHA-256(challenge + nonce_padded)[:4] == "0000"` (difficulty=4, ~1 in 65536)
3. Use `challenge + nonce_padded` as a Bearer token

```bash
python3 pricedancing-token.py rAAvxj forty-five-days
```

Valid periods: `twenty-four-hours`, `one-week`, `forty-five-days`, `one-year`, `four-years`, `all-time`.

## CSV Snapshots

These files are generated by `fetch-gold-prices.sh` and committed as snapshots.

| File | Source | Columns |
|---|---|---|
| `cafef-sjc.csv` | CafeF SJC gold bar history | `date`, `buy`, `sell` |
| `cafef-sjc-ohlcv.csv` | CafeF SJC (synthetic OHLCV) | `date`, `open`, `high`, `low`, `close`, `volume` |
| `cafef-ring.csv` | CafeF gold ring (Nhẫn ép vỉ Kim Gia Bảo) | `date`, `buy`, `sell`, `buy_change`, `sell_change` |
| `cafef-ring-ohlcv.csv` | CafeF gold ring (synthetic OHLCV) | `date`, `open`, `high`, `low`, `close`, `volume` |
| `sjc-today.csv` | SJC official API (today, all branches) | `date`, `branch`, `buy`, `sell`, `buy_diff`, `sell_diff` |
| `sjc-batch.csv` | SJC official API (2016–now, Hồ Chí Minh branch) | `date`, `branch`, `buy`, `sell`, `buy_diff`, `sell_diff` |

All prices are in full VND (e.g., `169500000.0` = 169,500,000 VND).

## Known Issues

**SJC API `buy_diff` / `sell_diff` are diffs against today's price, not daily changes.** The `BuyDifferValue` and `SellDifferValue` fields represent `price_on_date - price_today`. For example, on 2016-01-01 the buy price was 32,550,000 VND and today's is 169,500,000 VND, so `buy_diff = 32,550,000 - 169,500,000 = -136,950,000`. Today's own rows correctly show `0.0`. This means the diff values shift every day as the current price changes, making them unsuitable for historical analysis unless you know the reference date.

**Recommendation:** If you need daily price changes, compute them locally by comparing consecutive rows' `buy`/`sell` columns.
